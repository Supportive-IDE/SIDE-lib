SIDE-lib Basic Usage V3

The library's two main functions (parse() and feedback()) take a string of source code as input e.g.:


        
feedback(python_str, includeParseOutput = false, parseOptions = {showSymptoms=true, showMisconceptions=true, showCounterSymptoms=false, showConcepts=false})

        
...where python_str is a string of Python code. For best results, this should be a complete program rather than a snippet.

        
feedback() returns a JSON object containing basic information to provide feedback on a detected misconception indicator. E.g.:

        

        
The extendedFeedbackParams property on each feedback object contains more details about the indicator in URL parameter format. The intention is 
        that this information can be used to populate extended online guidance for an indicator. Our feedback implementation is hosted at https://supportive-ide.github.io/symptom-feedback. 
        To use this existing guidance, append the URL parameters to our host name e.g.:

        

        
If includeParseOutput is true, the object will also include the results of the parse() function below.

        
The optional parseOptions parameter can be used to configure parse output, if it is included.

        
parse(python_str, {showSymptoms=true, showMisconceptions=true, showCounterSymptoms=false, showConcepts=false} = {})

        
...where python_str is a string of Python code. For best results, this should be a complete program rather than a snippet.

        
parse() returns a JSON object containing information about detected symptoms and misconceptions. The third, optional parameter can be used to configure the output to include counter symptoms and concept indicators. In this example, the default settings are used:
